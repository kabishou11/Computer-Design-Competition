# 第三章：答辩技巧

## 3.1 答辩准备

### 3.1.1 答辩材料清单

| 材料 | 说明 | 注意事项 |
|-----|------|---------|
| **PPT** | 核心展示材料 | 15-20页为佳 |
| **演示视频** | 备用方案 | 3-5分钟 |
| **项目代码** | 现场演示 | 确保可运行 |
| **演示环境** | 本地或云端 | 准备网络备份 |
| **技术文档** | 详细说明 | 打印备用 |
| **常见问题** | 预设Q&A | 团队演练 |

### 3.1.2 PPT结构设计

```
答辩PPT结构：

1. 封面（1页）
   - 项目名称
   - 团队名称
   - 成员信息

2. 项目背景（1-2页）
   - 痛点问题
   - 现有方案不足
   - 我们的目标

3. 解决方案（2-3页）
   - 核心思路
   - 技术选型
   - 创新点

4. 技术实现（3-5页）
   - 系统架构
   - 核心算法
   - 关键技术

5. 功能展示（3-5页）
   - 演示截图
   - 交互流程
   - 效果对比

6. 项目成果（1-2页）
   - 数据指标
   - 用户反馈
   - 获奖情况

7. 总结展望（1页）
   - 项目总结
   - 未来计划

8. 致谢（1页）
   - 感谢评委
   - 团队信息
```

### 3.1.3 PPT制作要点

**视觉设计**：
- 简洁为主，避免花哨
- 统一配色和字体
- 图多字少，突出重点
- 关键数据突出显示

**内容原则**：
- 突出创新点和难点
- 技术细节适度
- 用数据和案例说话
- 准备备选详版

## 3.2 演讲技巧

### 3.2.1 时间分配

| 环节 | 时间 | 内容 |
|-----|------|------|
| 演示 | 5-8分钟 | 核心功能和亮点 |
| 讲解 | 5-8分钟 | 技术方案和创新 |
| Q&A | 5-10分钟 | 评委提问 |

### 3.2.2 演讲节奏

```
开场（30秒）：
- 用一个吸引人的案例开场
- 快速引入问题

主体（8-10分钟）：
- 层层递进，逻辑清晰
- 每个部分有小结
- 过渡自然

结尾（30秒）：
- 总结核心亮点
- 强调应用价值
```

### 3.2.3 语言表达

**推荐用语**：
- "我们通过...解决了..."
- "核心创新在于..."
- "实验结果表明..."
- "与现有方案相比..."

**避免用语**：
- "这个...那个..."（用词要准确）
- "可能...大概..."（要肯定）
- "我不太确定..."（要自信）

## 3.3 演示技巧

### 3.3.1 演示准备

```
演示清单：
□ 演示账号已登录
□ 所有页面已打开
□ 备用数据已准备
□ 错误处理已准备
□ 录屏已打开
□ 计时已准备
```

### 3.3.2 演示流程

1. **开场**
   ```
   "接下来请看我们的系统演示"
   ```

2. **核心功能**
   ```
   "首先展示核心功能..."
   ```

3. **亮点展示**
   ```
   "这是我们的创新亮点..."
   ```

4. **效果展示**
   ```
   "我们来看实际效果..."
   ```

5. **容错演示**
   ```
   "即使遇到异常情况..."
   ```

### 3.3.3 应对演示意外

| 意外情况 | 应对策略 |
|---------|---------|
| 程序崩溃 | 切换到视频演示 |
| 网络问题 | 演示离线功能 |
| 演示卡顿 | 跳过非核心功能 |
| 鼠标失灵 | 使用键盘快捷键 |

## 3.4 Q&A应对

### 3.4.1 常见问题类型

| 类型 | 示例 | 回答策略 |
|-----|------|---------|
| **技术细节** | "为什么选择这个算法？" | 准备技术选型理由 |
| **可行性** | "准确率能达到多少？" | 准备测试数据 |
| **创新性** | "和XX项目有什么不同？" | 强调差异化优势 |
| **局限性** | "有什么不足之处？" | 诚实承认，说明改进 |
| **扩展性** | "如何推广应用？" | 说明应用场景 |

### 3.4.2 回答技巧

**确认问题**：
```
"请问您是指..."
"我理解您的问题是..."
```

**结构化回答**：
```
"这个问题我们从三个方面考虑：
第一，...
第二，...
第三，...
"
```

**不会回答时**：
```
"这个问题我们确实没有深入考虑，
感谢您的建议，我们会后续改进。"
```

**遇到挑战时**：
```
"您提出的这个问题很好，
实际上我们是这样考虑的...
或者我们可以这样解决..."
```

## 3.5 团队配合

### 3.5.1 分工建议

| 角色 | 职责 | 人数 |
|-----|------|------|
| 主讲 | PPT讲解、核心演示 | 1人 |
| 副讲 | 技术细节补充 | 1人 |
| 操作 | 现场演示、代码展示 | 1人 |
| 后勤 | 材料准备、设备检查 | 0-1人 |

### 3.5.2 配合技巧

- 主讲全程把控节奏
- 副讲适时补充
- 操作提前准备好环境
- 提前演练分工配合
